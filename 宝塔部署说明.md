# DL Engine 官方网站 - 宝塔面板部署说明

## 📋 项目概述

**项目名称**: DL Engine 官方网站  
**技术栈**: Next.js 16 + React 19 + TypeScript + Tailwind CSS  
**部署方式**: 静态站点生成 (SSG)  
**多语言支持**: 中文/英文  

## 🛠️ 环境要求

### 服务器环境
- **操作系统**: CentOS 7+/Ubuntu 18.04+
- **内存**: 最低 2GB，推荐 4GB+
- **硬盘**: 最低 10GB 可用空间
- **Node.js**: 已确认宝塔面板支持 v24.10.0 ✅

### 软件版本
- **Node.js**: 18.17.0+ (推荐 20.x 或 24.x)
- **Nginx**: 1.18.0+
- **PM2**: 5.3.0+ (进程管理)

### 项目依赖版本
```json
{
  "next": "^16.2.1",
  "react": "^19.2.4", 
  "typescript": "^5.9.3",
  "tailwindcss": "^4.2.2",
  "next-intl": "^4.8.3"
}
```

## 📦 宝塔面板配置

### 1. 安装宝塔面板
```bash
# CentOS
curl -sSO http://download.bt.cn/install/install_6.0.sh && bash install_6.0.sh

# Ubuntu
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
```

### 2. 安装必要软件
在宝塔面板中安装以下软件：
- **Nginx 1.22+**
- **Node.js 版本管理器** (推荐 Node.js 20.x 或 24.x)
- **PM2 管理器**

### 3. 创建网站
1. 进入宝塔面板 → 网站 → 添加站点
2. 域名: `www.dlengine.com` (可添加其他域名)
3. 根目录: `/www/wwwroot/dlengine.com`
4. PHP版本: 纯静态 (无需PHP)
5. 创建数据库: 不需要 (静态站点)

## 🚀 部署步骤

### 步骤1: 上传项目文件
```bash
# 通过宝塔文件管理器或FTP上传项目文件到网站根目录
cd /www/wwwroot/dlengine.com

# 项目结构
├── app/                 # Next.js App Router 页面
├── components/          # React 组件
├── data/               # 静态数据
├── public/             # 静态资源
├── lib/                # 工具库
├── package.json        # 依赖配置
└── next.config.mjs     # Next.js 配置
```

### 步骤2: 安装依赖
```bash
# 进入项目目录
cd /www/wwwroot/dlengine.com

# 安装依赖
npm install --production

# 或使用淘宝镜像加速
npm install --production --registry=https://registry.npmmirror.com
```

### 步骤3: 构建项目
```bash
# 执行构建命令
npm run build

# 构建完成后会生成 .next 目录和静态文件
```

### 步骤4: 配置 PM2 进程管理

#### 创建 PM2 配置文件 `ecosystem.config.js`:
```javascript
module.exports = {
  apps: [{
    name: 'dlengine-website',
    script: 'npm',
    args: 'start',
    cwd: '/www/wwwroot/dlengine.com',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    error_file: '/www/wwwlogs/dlengine-error.log',
    out_file: '/www/wwwlogs/dlengine-out.log',
    log_file: '/www/wwwlogs/dlengine-combined.log',
    time: true
  }]
};
```

#### 启动 PM2:
```bash
# 启动应用
pm2 start ecosystem.config.js

# 设置开机自启
pm2 startup
pm2 save
```

### 步骤5: 配置 Nginx 反向代理

在宝塔面板中编辑网站配置文件：

```nginx
server {
    listen 80;
    server_name www.dlengine.com dlengine.com;
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|webp)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # Next.js 应用代理
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    
    # 启用 Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript 
               application/javascript application/xml+rss 
               application/json;
}
```

### 步骤6: SSL 证书配置
1. 在宝塔面板 → SSL → Let's Encrypt
2. 选择域名: `www.dlengine.com` 和 `dlengine.com`
3. 申请免费 SSL 证书
4. 开启强制 HTTPS

## 🔧 常见问题解决

### 问题1: 端口被占用
```bash
# 检查端口占用
netstat -tulpn | grep :3000

# 杀死占用进程
kill -9 <PID>

# 或修改 PM2 配置中的端口号
```

### 问题2: 内存不足
```bash
# 查看内存使用
free -h

# 优化 Node.js 内存使用
NODE_OPTIONS="--max-old-space-size=2048" npm run build
```

### 问题3: 构建失败
```bash
# 清理缓存重新构建
rm -rf .next node_modules package-lock.json
npm install
npm run build
```

### 问题4: 静态资源404
- 检查 `public` 目录权限
- 检查 Nginx 静态资源配置
- 确认文件路径正确

## 🔄 更新部署流程

### 常规更新
```bash
# 1. 备份当前版本
cp -r /www/wwwroot/dlengine.com /www/backup/dlengine-$(date +%Y%m%d)

# 2. 上传新版本文件
# 3. 安装依赖
npm install --production

# 4. 重新构建
npm run build

# 5. 重启应用
pm2 restart dlengine-website
```

### 紧急回滚
```bash
# 恢复备份版本
cp -r /www/backup/dlengine-20240330/* /www/wwwroot/dlengine.com/

# 重启应用
pm2 restart dlengine-website
```
