# 项目上下文协议 (Project Context Protocol)

## 项目概述
- **项目名称**: DL Engine 官方网站
- **项目类型**: 企业官网 (Next.js + TypeScript + Tailwind CSS)
- **核心功能**: 展示公司产品、技术实力，支持中英文双语
- **数据源**: **静态内容架构** (已放弃 WordPress)

## 架构变更说明
**重大变更**: 项目已完全放弃 WordPress 后端，转向纯前端静态架构

### 变更原因
- 更好的性能表现（无需 API 调用）
- 更简化的部署流程
- 更高的安全性（无数据库依赖）
- 更快的开发迭代速度

### 新架构特点
- **纯前端架构**: 所有内容直接在代码中管理
- **静态生成**: 使用 Next.js 的静态生成功能
- **本地化内容**: 中英文内容直接在代码中维护
- **现代化设计**: 采用最新的设计趋势和技术栈

## 技术栈要求
- **前端框架**: Next.js 14+ (App Router)
- **样式框架**: Tailwind CSS
- **开发语言**: TypeScript
- **多语言**: 国际化架构 (i18n)

## 核心需求
1. 中英文双语架构（静态内容管理）
2. 现代化、科技感的 UI 设计
3. Hero 区域和产品展示区域

## 项目约束
- 必须支持响应式设计
- 代码质量要求高，遵循 TypeScript 最佳实践
- 性能优化要求（SEO 友好）
- **内容更新需通过代码部署**

## 已完成的页面
- ✅ 首页 (中文版)
- ✅ DLE20 页面
- ✅ 英文版首页 (/en)

## 内容管理策略
- **产品信息**: 直接在组件中维护
- **公司介绍**: 静态内容文件
- **多语言内容**: 通过国际化配置管理
- **图片资源**: 本地存储或 CDN

- **静态站点生成** (SSG)
