# 项目结构文档

## 目录树
```
dlengine.com/
├── app/                          # Next.js App Router 目录
│   ├── page.tsx                  # 中文首页 (/)
│   ├── en/                       # 英文路由目录
│   │   └── page.tsx              # 英文首页 (/en)
│   ├── globals.css               # 全局样式
│   └── layout.tsx                # 根布局
├── components/                   # 可复用组件
│   ├── ui/                       # 基础 UI 组件
│   │   ├── Button.tsx
│   │   ├── Card.tsx
│   │   └── Header.tsx
│   ├── layout/                   # 布局组件
│   │   ├── Navbar.tsx
│   │   ├── Footer.tsx
│   │   └── Hero.tsx
│   └── sections/                 # 页面区块组件
│       ├── ProductShowcase.tsx
│       └── LatestArticle.tsx
├── lib/                          # 工具库和配置
│   └── i18n/                     # 国际化配置
│       ├── hooks.ts             # 国际化 hooks
│       └── config.ts            # 国际化配置
├── public/                       # 静态资源
│   ├── images/
│   └── icons/
├── specs/                        # 项目规范文档
├── docs/                         # 开发文档
└── middleware.ts                 # 中间件（语言检测）
```

### 新增的目录和文件
- ✅ `lib/i18n/config.ts`: 国际化配置常量
- ✅ 静态内容管理：所有内容直接在组件中维护

## 多语言路由架构

### 路径规范
- **中文路径**: 无前缀 (SEO 友好)
  - 首页: `/`
  - 关于页: `/about`
  - 产品页: `/products`

- **英文路径**: 带 `/en` 前缀
  - 首页: `/en`
  - 关于页: `/en/about`
  - 产品页: `/en/products`

### 语言检测逻辑
- 中间件自动检测浏览器语言偏好
- 英文用户自动重定向到 `/en` 路径
- 中文用户直接访问无前缀路径
- 支持手动语言切换

## 静态内容管理架构

### 内容存储策略
- **页面内容**: 直接在页面组件中维护
- **产品信息**: 在对应的产品组件中管理
- **多语言内容**: 通过国际化 hooks 和配置管理
- **图片资源**: 本地 `public/images/` 目录或 CDN

### 内容更新流程
1. **开发环境修改**: 直接在代码中更新内容
2. **版本控制**: 通过 Git 管理内容变更
3. **部署发布**: 自动构建和部署静态站点

## 组件架构

### 布局组件 (`components/layout/`)
- `Navbar.tsx`: 导航栏（支持中英文切换）
- `Footer.tsx`: 页脚
- `Hero.tsx`: Hero 区域

### 区块组件 (`components/sections/`)
- `ProductShowcase.tsx`: 产品展示区域
- `LatestArticle.tsx`: 最新内容区域（静态内容）

### UI 组件 (`components/ui/`)
- 基础 UI 组件，支持复用和定制

## 国际化架构

### 配置管理 (`lib/i18n/`)
- `config.ts`: 语言配置常量
- `hooks.ts`: 国际化 hooks

### 功能特性
- 路径前缀管理（中文无前缀，英文 `/en`）
- 语言检测和自动重定向
- 静态内容翻译
- SEO 友好的 URL 结构

## 部署架构

### 构建流程
1. **静态生成**: Next.js 预渲染所有页面
2. **内容优化**: 图片优化、代码压缩
3. **CDN 部署**: 静态资源分发

### 环境配置
- **开发环境**: 本地开发服务器
- **生产环境**: 静态站点托管（Vercel、Netlify 等）
