# 技术栈文档

## 核心框架
- **Next.js 14+**: 使用最新的 App Router 架构，支持服务端渲染和静态生成
- **TypeScript**: 提供类型安全，提高代码质量和开发效率
- **Tailwind CSS**: 原子化 CSS 框架，实现快速样式开发

## 开发工具
- **Node.js**: 运行时环境 (推荐版本 18+)
- **npm**: 包管理器

## 数据层
- **静态内容管理**: 所有内容直接在代码中维护，无需外部数据源
- **本地化内容**: 中英文内容通过国际化配置管理

## 多语言支持
- **自定义国际化架构**: 基于路径的多语言路由 (/en 为英文，/ 为中文)
- **中间件支持**: 自动语言检测和重定向
- **静态内容翻译**: 内容直接在组件中维护

## 开发依赖
- **ESLint**: 代码质量检查
- **Prettier**: 代码格式化
- **TypeScript ESLint**: TypeScript 专用规则

## 构建和部署
- **Vercel**: 推荐部署平台（Next.js 官方平台）
- **静态站点生成** (SSG): 预渲染所有页面
- **环境变量管理**: 支持开发、测试、生产环境配置


### 新架构优势
- ✅ **性能优化**: 无需 API 调用，页面加载更快
- ✅ **简化部署**: 纯静态站点，部署更简单
- ✅ **内容可控**: 所有内容在代码中管理，版本可控
- ✅ **安全性高**: 无数据库依赖，攻击面更小

## 技术栈演进
从最初的 WordPress + GraphQL 架构演进为纯前端静态架构，主要变化：
1. **数据源**: WordPress API → 静态内容
2. **多语言**: WPML + next-intl → 自定义国际化
3. **部署**: 动态渲染 → 静态生成
4. **开发**: API 集成 → 纯前端开发

这种架构更适合企业官网场景，提供了更好的性能和更简单的维护流程。
