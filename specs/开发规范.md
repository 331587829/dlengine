# 开发规范文档

## 代码风格

### TypeScript 规范
- 使用严格模式 (`strict: true`)
- 禁止使用 `any` 类型
- 函数返回值必须明确指定类型
- 接口命名使用 PascalCase，属性使用 camelCase

### React 组件规范
- 使用函数式组件 + Hooks
- 组件文件名使用 PascalCase
- Props 使用 TypeScript 接口定义
- 避免在组件内直接使用内联样式

### 文件命名
- 组件文件：`PascalCase.tsx`
- 工具函数：`camelCase.ts`
- 配置文件：`kebab-case.config.ts`
- 类型定义：`PascalCase.ts`

## Git 提交规范

### 提交信息格式
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### 提交类型
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

## 目录结构规范

### 组件组织原则
- 单一职责：每个组件只负责一个功能
- 可复用性：通用组件放在 `components/ui/`
- 页面特定组件放在 `components/sections/`

### 文件导入顺序
1. React 和 Next.js 导入
2. 第三方库导入
3. 本地组件导入
4. 样式导入
5. 类型导入

## 样式规范

### Tailwind CSS 使用
- 优先使用 Tailwind 类名
- 避免自定义 CSS，除非必要
- 使用响应式设计前缀（`sm:`, `md:`, `lg:`）
- 颜色使用项目设计系统中的颜色

### 响应式设计
- 移动优先：先写移动端样式，再添加响应式断点
- 断点标准：sm (640px), md (768px), lg (1024px), xl (1280px)

## 数据获取规范

### GraphQL 查询
- 所有查询放在 `lib/graphql/queries/` 目录
- 查询使用命名导出
- 错误处理使用 try-catch 块
- 使用 TypeScript 类型确保数据安全

### 环境变量
- 敏感信息使用环境变量
- 环境变量前缀：`NEXT_PUBLIC_` 用于客户端变量
- 创建 `.env.example` 文件记录所有需要的环境变量

## 性能优化规范

### 图片优化
- 使用 Next.js Image 组件
- 指定图片尺寸和加载策略
- 使用 WebP 格式优先

### 代码分割
- 使用动态导入 (`import()`) 分割大组件
- 路由级别代码分割自动处理

### SEO 优化
- 每个页面设置合适的 meta 标签
- 使用语义化 HTML 标签
- 确保页面可访问性

## 测试规范

### 单元测试
- 关键组件编写单元测试
- 测试文件放在 `__tests__/` 目录
- 使用 Jest 和 React Testing Library

## 部署规范

### 构建检查
- 构建前运行 TypeScript 类型检查
- 构建前运行 ESLint 检查
- 确保所有测试通过

### 环境配置
- 区分开发、测试、生产环境
- 环境变量在不同环境正确配置
- 使用 Vercel 平台进行部署
